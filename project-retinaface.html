01
Problem

경량화의 딜레마: 속도 vs. 정확도

RetinaFace는 RetinaNet 기반의 얼굴 탐지 SOTA 모델입니다. 아키텍처 분석을 통해, Detection Head에 연산 부담이 집중되며 경량 백본(MobileNet v1) 도입 시 성능이 약 13%나 급감하는 구조적 원인을 파악했습니다.

  • Edge Device에서의 실시간 구동 제약 (낮은 FPS)
  • Small Face에 대한 탐지 정확도 유지의 기술적 한계
  • 경량화 시 발생하는 급격한 성능 저하

RetinaNet Detection Head 연산 비중 분석

02
Approach

4가지 아키텍처 개선 모듈 직접 설계 및 이식

관련 논문을 역추적하여 문제의 원인을 정의하고, 이를 해결하는 4개의 커스텀 레이어를 직접 설계하여 PyTorch 코드베이스에 이식했습니다.

Module 1

ECA-CBAM

중요 채널/공간 정보를 강조. CBAM의 MLP를 1D Conv(ECA)로 대체해 연산 부담을 줄이면서 표현력을 유지합니다.

Module 2

WFPN

FPN의 단순 sum 대신 학습 가능한 가중치 결합을 도입. 스케일별 중요도를 학습하여 정보 손실을 줄입니다.

Module 3

SCM (Shuffle Context Module)

채널 수를 줄여 연산을 억제한 뒤 채널 셔플로 컨텍스트를 혼합하여 효율적으로 표현력을 강화합니다.

Module 4

Deformable Convolution

Offset 기반 sampling으로 소형·변형 얼굴에서도 안정적으로 특징을 포착하여 탐지 성능을 보완합니다.

특히 Experiment 2 항목의 비교 테이블은 수치가 명확하고 강조 행이 잘 표시되어 있어 전달력이 높습니다.

## Crop for Trust 페이지 (`project-crop-for-trust.html`)

### ✅ 잘 된 점
- Experiment 2 테이블 : D_GAC / D_blur / D_black / D_crop을 나란히 보여주는 구성이 매우 효과적입니다.
- 헤더 수치가 Quick Stats 4개로 간결하게 정리되어 있습니다.
- 데이터 변환 4단계 이미지를 grid로 나란히 배치한 것이 직관적입니다.

### ⚠️ 개선 제안

1. **Hero 수치 정확성 확인 필요**
   - 현재 Hero에 `97.46%` 가 "YOLO-Crop Acc."로, `93.40%`가 "Hybrid Acc."로 표시되어 있습니다.
   - 그러나 이 두 수치의 맥락이 다릅니다. YOLO-Crop Acc.는 배경 제거 비교 실험(Exp2)에서 YOLO-Crop 단독 데이터셋의 정확도이고, Hybrid Acc.는 최종 파이프라인의 정확도입니다.
   - 열람자가 "YOLO-Crop이 더 높으면 왜 Hybrid를 씀?" 하고 의문을 가질 수 있습니다.
   - **제안:** Hero에는 "+4.69% (Hybrid vs. GAC Baseline)" 처럼 핵심 주장 중심의 수치만 남기거나, 97.46%는 제거하거나 각주 설명을 추가하는 것이 좋습니다.

2. **이미지 fig 파일들의 파일명에서 실제 이미지 폴더명 경로 확인 필요**
   - `src="Crop for Trust 프로젝트 이미지/fig1.png"` 형식이 사용되고 있습니다.
   - 실제 폴더 내에는 `문제 정의 - 마킹 이미지.jpg`, `results - 정확도 비교 이미지.jpg` 등 원래 파일명이 남아있습니다.
   - fig1~fig11로 파일을 실제로 rename했는지 확인이 필요합니다. rename하지 않았다면 이미지가 화면에 표시되지 않을 수 있습니다.
   - **제안:** 실제 파일명과 HTML의 src 경로가 일치하는지 브라우저 개발자 도구(F12 → Console)에서 확인하세요.

3. **Experiment 3 이미지 2개의 설명 명확성**
   - `fig9.png`는 "Original 데이터에 대한 Attention Alignment", `fig10.png`는 "GAC 데이터에 대한 Attention Alignment"로 표시되어 있습니다.
   - 메시지는 "YOLO-Crop이 Attention을 정렬했다"인데, Before/After를 "Original vs. YOLO-Crop"으로 레이블링하면 더 직접적입니다.
   - **제안:** 이미지 캡션을 "Before: Original 데이터의 Grad-CAM (배경 분산)" / "After: YOLO-Crop 적용 후 Grad-CAM (손상 집중)"으로 수정하는 것을 고려하세요.

4. **CNN 모델 실험 범위 카드 제거됨**
   - 이전 버전에 있던 "실험 대상 모델" 요약 박스(VGG16, ResNet152, MobileNet 목록)가 사라진 것으로 보입니다.
   - 이 정보는 "다양한 모델로 일반성을 검증했다"는 신뢰성을 높이는 내용이므로, 짧게라도 유지하는 것이 좋습니다.
   - **제안:** Methodology 섹션 하단에 한 줄 텍스트 또는 작은 뱃지 형태로 실험 모델 목록을 다시 포함시키세요.

## 화성시 페이지 (`project-hwaseong.html`)

### ✅ 잘 된 점
- 6개 섹션의 흐름(문제 → Stage 1 → Stage 2 → 결과 → 정책 → 산출물)이 자연스럽게 연결됩니다.
- FRI 공식을 `0.5 × 위해지표 + 0.2 × 취약지표 − 0.3 × 경감지표` 로 시각화한 코드 블록이 인상적입니다.
- T-MAP API를 사용해 단순 직선 거리가 아닌 실제 도로망 거리를 계산했다는 점을 별도 Step으로 강조한 것이 좋습니다.

### ⚠️ 개선 제안

1. **Hero 통계 수치 구체성 강화**
   - 현재 Hero Stats가 `대상`, `8,009건`, `12곳`, `T-MAP`으로 구성되어 있습니다.
   - `T-MAP`은 사용한 도구(Tool)이므로 성과(Impact)와 성격이 다릅니다.
   - **제안:** `T-MAP` 자리를 `5개소` (취약 구역 최종 도출 수) 또는 `3개 단지` (Top 3 선정) 같은 결과 수치로 교체하는 것을 권장합니다.

2. **12개 산업단지 위험도 차트 Placeholder 유지 여부 확인**
   - 현재 "12개 산업단지 위험도 점수 비교 차트" placeholder가 Stage 1 하단에 있습니다.
   - 해당 이미지가 완성되면 반드시 채워주세요. 빈 placeholder는 미완성인 인상을 줍니다.
   - **제안:** 이미지가 없다면 placeholder 대신 텍스트 기반 순위 리스트나 간단한 수치 카드를 활용하는 것도 하나의 방법입니다.

3. **결과 섹션 — "향남제약산업단지 한정"이라는 맥락 제공**
   - 현재 04. Results 섹션은 향남제약산업단지를 기준으로 설명하고 있는데, 이 사실을 섹션 첫 줄에서만 짧게 언급합니다.
   - 독자 입장에서 "다른 2개 단지는 어떻게 됐나?" 하는 의문이 생길 수 있습니다.
   - **제안:** 결과 섹션 상단에 "이하의 내용은 Top 3 중 향남제약산업단지를 사례로 한 정밀 분석 결과입니다. 나머지 2개 단지에 대해서도 동일한 방법론이 적용되었습니다." 같은 짧은 안내 문구를 추가하면 독자의 혼란을 줄일 수 있습니다.

4. **활동 사진 이미지 미사용**
   - `화성시 프로젝트 이미지/활동 사진.png` 파일이 이미지 폴더에 있지만, 현재 페이지에서 사용되지 않고 있습니다.
   - **제안:** 06. Output & Award 섹션에 시상식 또는 발표 현장 사진으로 추가하면 실질적인 수상 증거를 보여줄 수 있어 신뢰도가 높아집니다.

## 공통 제안

1. **이미지 로딩 실패 시 alt 텍스트 개선**
   - 현재 alt 텍스트가 "Marking Example", "GAC Validation" 등 영문으로 짧게 작성되어 있습니다.
   - 이미지 로딩이 실패하면 alt 텍스트가 표시되므로, "GAC 성능 검증: Scatter Plot (Edge L1 vs. Feature Map Distance)" 처럼 내용을 설명하는 텍스트가 더 적합합니다.

2. **모바일 반응형 점검**
   - 아키텍처 비교 테이블이나 `grid-cols-4` 레이아웃은 모바일에서 가로 스크롤이 생길 수 있습니다.
   - 브라우저 개발자 도구에서 모바일 뷰(375px)로 확인하는 것을 권장합니다.

3. **Dark Mode에서 img-placeholder 가시성**
   - Dark Mode 적용 시 dashed 테두리 색상이 `#334155`로 설정되어 있어 배경과 대비가 낮을 수 있습니다.
   - `border-color: #4f6182` 정도로 조금 더 밝게 조정하면 가독성이 향상됩니다.

# Task.
- Face Detection 모델인 RetinaFace의 Architecture를 개선하여 성능을 높이는 것.
- 더욱 자세한 목표는 연산량 증가를 최소화하면서 성능을 높이는 것임. 즉, 연산량 대비 성능 증가폭을 최대화하는 것.

# Problem Definition.
## 경량화 시 성능이 급감하며, 성능 격차가 심함.
- 기존 RetinaFace의 경우, ResNet-152 모델과 MobileNet-v1 모델을 백본으로 활용하는데 경량화를 원하는 경우 MobileNet v1을 활용하는 것으로 보임.
- 다만, 백본을 MobileNet v1으로 변경시 ResNet-152를 사용했을 때보다 성능이 약 13 AP 급감함. 이에, 우리는 MobileNet-v1을 백본으로 활용하면서 AP 성능을 올리고 싶음.

## Model Architecture의 최적화 가능성.
- RetinaFace의 기본 구조는 RetinaNet과 매우 유사함. RetinaNet에서 Detection Head를 통해 Loss가 연결되는 부분에 Face Detection Loss를 연결해주면 RetinaFace임.
- 따라서, RetinaFace의 병목 구조를 살펴보기 위하여, RetinaNet의 경량화 후속 논문들을 살펴봄. (Light-Weight RetinaNet for Object Detection, https://arxiv.org/abs/1905.10011)
- 해당 논문을 보면, RetinaNet의 레이어별 Flops / Memory 할당량을 보여줌. 이 결과를 살펴보면, Detection Head 초기 부분에 굉장히 많은 Flops가 할당됨을 확인할 수 있음.
- 따라서, 우리는 모델 구조를 개선하여, 중간 레이어에서 정보 전달을 원활하고 최적화해보려고 함.

# Method.
## Overall Architecture.
- 기존 구조: Backbone (MobileNet v1) -> FPN -> SSH (Context Module) -> Detection Head.
- 우리 구조: Backbone (MobileNet v1) -> ECA-CBAM -> WFPN (FPN -> WFPN으로 대체) -> ECA-CBAM -> SCM (SSH -> Shuffle Channel Module로 대체) -> ECA-CBAM -> Detection Head.
- 추가로, 우리 구조에서는 모든 Convolution을 Deformable Convolution으로 대체함.

## FPN -> WFPN.
- 기존 FPN은 단순히 Feature Pyramid를 고수준의 정보를 저수준의 정보로 흘려주는 것이었고, 연산으로 보면 sum 이었음.
- WFPN은 이 과정에서 Leanable Parameter를 하나 추가하여, 고수준의 정보와 저수준의 정보를 단순히 sum하는 것이 아닌 중요도(가중치)에 따라 합하도록 설정함.
- 이를 통해, 정보 손실을 줄이고 성능을 향상시킬 수 있음.

## ECA-CBAM 추가.
- 정보 흐름을 원활하게 하고 최적화하기 위해 최신 매커니즘인 Attention 모듈을 추가함. 그 중에서 CBAM 모듈을 활용하며 이는 중요한 채널/공간 정보를 강조하여 Detection Head에 전달되는 입력을 정제하기 위함임.
- 우리는 기존 CBAM 구조에서 Channel Attention에 포함되는 MLP Layer를 1D Convolution으로 대체함. 이렇게 설계한 이유는, 우리는 연산량 대비 최대 성능 gain을 얻기 위함이기 때문에 연산 부담을 줄이면서 표현력을 유지하기 위함임.
- CBAM을 선택한 이유는 공간/채널별 Attention이 가능하며, Adaptive Kernel size를 활용하여 연산이 효율적이기 때문임.

## SSH -> SCM.
- 기존 SSH는 3x3, 5x5, 7x7 Convolution을 통해 다양한 receptive field를 추출하고, 이를 결합하는 과정을 거침.
- 반면, 우리의 SCM은 3x3, 5x5, 7x7, 10x10 Convolution을 통해 더 다양한 receptive field를 추출하지만, 채널 수를 낮추어 연산량은 더 줄임. 또한, 결합 과정에서 다양한 receptive field의 결과를 합해주기 위해 Shuffle 과정을 거침.
- 이를 통해 연산량을 줄이면서 더욱 다양한 receptive field의 정보를 결합할 수 있게 됨.

## Deformable Convolution.
- 기존 Convolution을 Deformable Convolution으로 대체함. 이는 DCN (Deformable Convolutional Network)의 아이디어를 차용한 것임.
- Deformable Convolution은 기존 Convolution과 달리, offset 기반 sampling으로 소형/변형 얼굴에서도 안정적으로 특징을 포착할 수 있음.
- 따라서, 우리는 Deformable Convolution을 통하여, 성능을 더욱 개선하려고 노력함.

# Results.
- 우리의 구조는 WiderFace 데이터셋 기준으로 성능을 측정했을 때, 기존 RetinaFace보다 Hard Case에서 약 2.7 AP 향상됨.
- Flops는 기존 RetinaFace 대비 약 0.09G 증가로 실제 Edge Device에 도입시 미미한 수준으로 증가함.
- 결과를 보면, Easy Case에서는 +1 AP, Medium Case에서는 +1.7 AP, Hard Case에서는 +2.7 AP 증가하는 것으로 보아 우리의 Architecture는 작거나 변형되어 탐지가 어려운 환경에서 기존 RetinaFace와의 격차가 더욱 벌어지는 것을 확인할 수 있음.
- 우리는 이 결과를 Ablation Study를 통해 레이어별 성능 증가폭을 조사하고 표로 정리함.
- 또한, 실제 사진을 바탕으로 Detection 결과를 Bounding Box로 표시하여 정성적 평가를 수행함. 그 결과, 기존 RetinaFace가 놓친 객체에 대해 우리의 모델이 잘 Detection 해냄을 확인할 수 있음.
- 결과적으로, 이 프로젝트를 통해 저는 Capstone 수업에서 1등을 하고, A+ 학점을 취득할 수 있었습니다.
